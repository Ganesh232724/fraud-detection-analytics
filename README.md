# Fraud Detection Using Machine Learning

Fraud Detection Analytics is a business analytics project focused on identifying and preventing
fraudulent activities in financial transactions using data-driven insights and machine learning.
The system analyzes historical and real-time transaction data to detect anomalies, patterns, and
suspicious behavior that may indicate fraud.

## Key Objectives
- Detect fraudulent transactions using historical and real-time data
- Identify patterns and anomalies indicative of fraud
- Reduce financial losses and operational risks
- Provide actionable insights for prevention and mitigation

## Key Features
- **Real-Time Fraud Monitoring** — tracks transactions as they occur to flag suspicious activity
- **Anomaly Detection** — identifies deviations from normal transaction patterns
- **Risk Scoring** — assigns a 0–100 fraud risk score to each transaction
- **Alert Generation** — classifies transactions as APPROVE / MANUAL REVIEW / BLOCK

## Tech Stack
- Python (Pandas, NumPy, Scikit-learn)
- Matplotlib / Seaborn (visualization)
- SQL / NoSQL databases (production data storage)
- Power BI / Tableau (dashboards)

## Algorithm Overview
1. Data Collection
2. Data Cleaning
3. Exploratory Data Analysis (EDA)
4. Feature Engineering
5. Handling Class Imbalance
6. Train/Test Split (stratified, 80/20)
7. Model Training — Logistic Regression, Random Forest, Isolation Forest
8. Model Evaluation — Precision, Recall, F1, ROC-AUC
9. Risk Scoring — weighted ensemble (0–100 score)
10. Alert Generation
11. Deployment & Monitoring
12. Feedback Loop for retraining

Full details are documented in [`docs/Fraud_Detection_Analytics_Algorithm_Dataset.pdf`](docs/Fraud_Detection_Analytics_Algorithm_Dataset.pdf).

## Dataset
- **Source**: Internal transaction systems, or public benchmark datasets (e.g. Kaggle Credit Card
  Fraud Detection, IEEE-CIS Fraud Detection)
- **Records**: ~250,000–1,000,000+ transactions (typical prototype scale)
- **Target variable**: `is_fraud` (0 = legitimate, 1 = fraudulent)
- **Split**: 80% train / 20% test, stratified to preserve fraud ratio

## How to Run

```bash
# 1. Clone the repository
git clone https://github.com/<your-username>/fraud-detection-analytics.git
cd fraud-detection-analytics

# 2. Install dependencies
pip install -r requirements.txt

# 3. Run the pipeline
python fraud_detection_source.py
```

Running the script will:
- Generate/clean a transaction dataset
- Save an EDA report as `eda_report.png`
- Train Logistic Regression, Random Forest, and Isolation Forest models
- Print evaluation metrics (accuracy, precision, recall, F1, ROC-AUC)
- Save a fraud alert report to `fraud_alert_report.csv`
